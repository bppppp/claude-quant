## 3. 策略跨平台稳健性评估

### 3.1 评级：**较差 (Poor)**

该策略在跨平台迁移时表现出显著的绩效衰减，不具备生产环境所需的稳健性。

### 3.2 关键证据

| 证据 | 结论 |
|------|------|
| 年化收益差 30pp（10.78% vs 40.95%） | 策略 alpha 高度依赖数据源 |
| 日收益相关性仅 0.64 | 两个平台在约 1/3 的交易日做出相反方向押注 |
| JQ 最大回撤 2.5 倍于本地（-35.52% vs -14.16%） | 风控体系在跨平台时失效 |
| JQ 夏普 0.33 vs 本地 2.05 | JQ 承担了 1.35 倍波动，仅获 1/4 的收益 |
| 胜率差 9.3pp（38.2% vs 47.5%） | 每 10 笔交易中，JQ 比本地多亏 1 笔 |

### 3.3 差异根因（按影响程度排序）

**P0 — 前复权数据差异导致选股完全不同**

两个平台使用不同的数据供应商，前复权算法不同 → 同一交易日的价格序列不同 → 以下所有环节都会产生分歧：

```
复权价格 → 5日跌幅(drop_5d) → 信号触发(drop<-10%)
         → RSI(6)计算        → 信号触发(RSI<20)
         → 入场成本基准       → 止损-2%触发时机
         → 持仓MTM估值       → 回撤计算
```

对于均值回归策略（核心逻辑是"超跌反弹"），前复权决定了"谁看起来超跌了"——这是整个策略的基石。两个平台选出了不同的股票，导致后续一切指标发散。

**P1 — 股票池动态 vs 静态差异**

- JQ 使用实时指数成分股（HS300 每年调整 2 次，个股会调入/调出）
- 本地使用 `data/config.py` 的静态快照（2026-06-01 时点）
- 2018-2022 期间，HS300 成分股变动约 40-60 只，这些差异造成约 15% 的候选池不同

**P2 — RSI 实现差异**

- JQ: Wilder smoothing (`alpha=1/period, ewm(adjust=False)`)
- 本地: 金玥数据预计算的 RSI6（实现细节未知）

在 RSI < 20 的极端阈值下，微小计算差异就决定信号有无。

### 3.4 策略优点（跨平台一致性方面）

| 优点 | 证据 |
|------|------|
| 交易活跃度接近 | 3188 vs 3696 笔，量级一致 |

## 4. 建议

### 4.1 短期（提升跨平台一致性）

1. **统一 RSI 实现**：在本地用 Wilder smoothing 重算 RSI6，与 JQ 对齐
2. **对齐股票池**：从 JQ 导出每日实际成分股列表，替换本地静态快照
3. **输出每日信号日志**：两边同时输出 `{date, symbol, drop_5d, RSI6, triggered}` 做逐日对比
4. **抽样对比前复权价格**：选 10 只代表性股票，在关键信号日对比两个数据源的价格序列

### 4.2 长期（策略本身改进）

1. **降低对极端阈值的依赖**：`drop < -10% AND RSI < 20` 对数据过于敏感，考虑用排序法（横截面 rank）替代绝对阈值
2. **加入信号确认机制**：不只依赖单一价格快照，要求连续 2 日满足条件
3. **减少参数对数据源的敏感度**：用分位数阈值代替固定百分比阈值
