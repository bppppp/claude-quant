# A股做多交易策略生成 Prompt

# 角色
A 股量化策略专家，专注做多策略。

# 目标
- 熊市规避：下跌趋势仓位 ≤ 20%
- 震荡市盈利：高抛低吸、波段、均值回归
- 震荡下行不反复止损：持续 1 年以上缓慢阴跌中保持稳定

# 硬指标
| 指标 | 要求 |
|---|---|
| 年化收益 | ≥ 15% |
| 最大回撤 | ≤ 15% |
| 年化波动率 | ≤ 18% |
| 夏普 | ≥ 1.2 |
| 胜率 | ≥ 55% |
| 盈亏比 | ≥ 1.5 |
| 月胜率 | ≥ 60% |
| 相对沪深300 超额 | ≥ 10% |

# 策略框架（10 大模块）

## 1. 市场环境识别
- 4 状态：BULL / SIDEWAYS / BEAR / CRASH
- 状态判定规则（具体阈值）
- 状态确认机制（迟滞 + 冷却期）
- 输出每日确认状态和目标仓位

## 2. 参数体系
- 5 层结构：核心不可调 / 核心可调 / 状态特化 / 风格特化 / 因子特化
- 5 大自适应参数：vol / wr / breadth / holding / atr
- 参数扰动测试（±20%）
- 集中配置管理

## 3. 选股模型
- 4 类因子：趋势 / 动量 / 量价 / 形态，至少 12 个
- 因子预处理：去极值 → 标准化 → 中性化
- 因子合成：等权 / IC加权 / IR加权
- 因子权重随市场状态动态切换

## 4. 买点信号
- 至少 6 类：均线金叉、KDJ金叉、布林带、Donchian 突破、MACD 底背离、均线粘合突破
- 每类说明：触发条件、确认机制、假突破过滤

## 5. 卖点信号
- 4 类：均线死叉、KDJ超买、跌破布林中轨、MACD 顶背离
- 3 层风控：
  - 硬止损（仅 BULL 状态）
  - 移动止盈（核心：highest - k×ATR，只允许上移）
  - 时间止损
- 卖点优先级：CRASH 强制清仓 > 移动止盈 > 分批止盈 > 时间止损 > 硬止损

## 6. 震荡下行专项
- 4 维检测：60 日累计跌幅、低波动率、MA60 走平、MA20<MA60
- 专项规则与常规策略差异化：仓位 20% / 单只 3% / 持仓 ≤ 5 日 / 反弹 > 5% / 量比 1.5× / 浮盈 3% 减 1/3，5% 全走 / 禁用硬止损、突破、均值回归、网格
- 退出信号：累计收益转正、放量大阳线、8 周时间窗口

## 7. 回测
- 区间：2016-01-01 ~ 2025-12-31
- 基准：沪深300
- 初始资金：1000 万
- Walk-Forward：训练 3 年 → 测试 1 年，至少 5 轮
- 关键年份单独分析：2015 / 2016 / 2018 / 2020 / 2022 / 2023 / 2024 阴跌
- 指标：年化、最大回撤、夏普、卡玛、胜率、盈亏比、95% 置信区间
- 避免未来函数

## 8. 风控
- 回撤阶梯：5% 降至 70% / 10% 降至 40% / 15% 空仓 5 日 / 20% 暂停
- 资金管理：单只亏 > 3% 净值清仓 / 当日亏 > 2% 停开 / 月亏 > 5% 腰斩 / 季亏 > 10% 用 1/3 仓位 / 年亏 > 15% 空仓
- 告警：回撤、单日亏、CRASH、黑天鹅
- 监控看板：净值、持仓、暴露、回撤

## 9. 参数稳定性
- 扰动测试 ±20%
- 脆弱参数识别（扰动后回撤变化 > 30%）
- 加固方案：限幅、离散化、加约束

## 10. 失效应对
- 失效场景清单：风格切换、流动性危机、黑天鹅、监管变化
- 每个场景的应对方案
- 策略容量限制

# 关键机制

## 震荡下行
MA60 走平或微跌、每天小阴小阳但重心持续下移。

传统硬止损的问题：小反弹 → 突破买入，小下跌 → 硬止损卖出，反复几次本金亏光。

解决方向：状态级止损优先 / ATR 自适应 / 移动止盈 / 冷却期管理。

## 5 大自适应参数
- 止损倍数 k：随波动率缩放
- 持仓周期：随市场评分
- 仓位上限：随市场宽度
- 开仓阈值：随滚动胜率
- 因子权重：随市场状态

# 数据与实现
- 数据：日 K 线（akshare / baostock）
- 不使用：ML、HMM、基本面、北向资金、融资余额、宏观数据
- 实现：完整可运行的 Python 工程

# 输出格式

按以下顺序组织：

1. 策略概览（300 字内）
2. 市场环境识别（4 状态 + 确认机制）
3. 分状态参数表（BULL / SIDEWAYS / BEAR / CRASH 各一套）
4. 多层止损体系
5. 震荡下行专项
6. 5 层可调参数结构
7. 选股模型
8. 买卖信号
9. 回测结果
10. 参数稳定性测试
11. 代码工程
12. 风险声明（失效场景 + 容量 + 免责）
