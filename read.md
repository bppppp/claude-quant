## 2. 调优方向与结果分析

### 2.1 调优路径

| 版本 | 核心改动 | 主要效果 |
|------|---------|---------|
| V2 | 缩短 BULL k (3.0→2.5) + 减少 top_n (8→6) + 提高阈值 (0.70→0.75) + 降低分批止盈 (8%→5%) | 改善回撤（-15.29%→-13.76%），但年化持平 |
| V3 | 全部收紧：top_n 4 + k 2.0 + 阈值 0.80 + 持仓 10 天 | 调过头：年化恶化到 -7.51% |
| V4 | 部分回调 V3 + 新增 atr_k_min 1.5 | 改善盈亏比至 1.69，但胜率掉到 24% |
| V5 | 试探 V1-V4 中间点 | 表现最差：年化 -7.79%，回撤 -16.10% |

### 2.2 核心发现

#### 发现 1: BULL 状态是亏损主因
- V1 136 笔 BULL 累计 **-122.51%**
- V2-V4 各约 90-110 笔 BULL 累计 **-103% ~ -104%**
- V5 BULL 累计 **-138.98%**（最差）
- 移动止盈占 72-78%，但每次触发后都亏损

#### 发现 2: 调参空间已触底
- V1 → V2 改善 0.07pp
- V2 → V3 恶化 0.89pp
- V3 → V4 改善 0.55pp
- V4 → V5 恶化 0.83pp
- 调优方向来回震荡，**没有找到单调改进路径**

#### 发现 3: BULL 信号在 2018-2020.4 不友好
- BULL 状态出现时，往往已是反弹末期
- 中证 1000 整体下跌期间，BULL 状态识别滞后
- 移动止盈在低波动 + 下跌趋势中被反复触发

#### 发现 4: 2018-2020 区间本身不利
- 2018: 中证 1000 跌 -36%（贸易战）
- 2019: 反弹 +24%（震荡）
- 2020.Q1: 疫情暴跌 -8%
- 2020.Q2: 反弹 +15%
- **策略设计目标场景"震荡+熊市"，但实际是 2 年熊市 + 1 年震荡**

## 4. 5 个核心问题诊断

### 问题 1: BULL 状态识别不准
- **症状**: 状态切换有 3 日迟滞 + 5 日冷却期
- **影响**: BULL 信号在熊市反弹末才确认，错过底部
- **建议**: 缩短迟滞期（min_duration 3→2）或减少冷却期

### 问题 2: 移动止盈在低波中失效
- **症状**: 75% 卖出由移动止盈触发，但触发后都亏
- **影响**: 移动止盈在低波 + 慢熊中被反复打掉
- **建议**: 移动止盈 k 在低波环境下应增大（spec 已实现 vol_adj 但不够）

### 问题 3: 分批止盈阈值过高
- **症状**: BULL 8% 才减 1/3（spec）
- **影响**: 在中证 1000 平均日波 1.5% 下很难触发
- **建议**: BULL 第一档降到 4-5%（已在 V2-V4 试过）

### 问题 4: 单只上限数学约束过紧
- **症状**: BULL 8 只 × 11.25% = 90%，实际可操作空间小
- **影响**: 单笔 11.25% 仓位，下跌 -3% 就吃掉 0.34% 净值
- **建议**: 减少 top_n 到 4-5（已在 V3-V4 试过，但胜率下降）

### 问题 5: 因子选股对中证 1000 失效
- **症状**: BULL 累计 -103% 表明选出的股票整体跑输指数
- **影响**: 策略在熊市反弹中选出"假强势"股
- **建议**: 加入市场宽度（market_breadth）调整权重，或降低熊市反弹中 BULL 仓位

## 5. 对 ATOS 策略的看法与建议

### 5.1 策略定位评估

| 维度 | 评估 |
|------|------|
| **设计合理性** | 高。spec 状态识别 + 自适应参数 + 5 层风控的设计框架完整 |
| **回测实现** | 高。T+1 结算、涨跌停、停牌、100 整手、5 费 + 滑点都到位 |
| **中证 1000 适配** | 中。13 因子 + IC 加权在 2018-2020 表现一般 |
| **震荡市表现** | 待验证。V1-V5 期间 BULL 状态主导，未充分测试震荡市 |
| **风控有效性** | 高。最大回撤全部控制在 16% 内，未爆仓 |

### 5.2 spec 设计 vs 实际表现

| spec 章节 | 期望效果 | 实际表现 | 评估 |
|----------|---------|---------|------|
| §3 状态参数 | 不同状态匹配不同参数 | 实际 BULL 仍占主导 | 状态识别滞后 |
| §3 硬止损 -8% | BULL 保护 | V1 仅触发 1 次（不是"硬止损 -8%"原因） | 阈值过严 |
| §4.2 移动止盈 | 趋势跟踪 | 75% 卖出都触发 | 触发但亏 |
| §4.3 分批止盈 | 锁利 | 19% 卖出 | 触发少 |
| §6.1 自适应 | 动态调整 | 部分实现（持仓周期、阈值） | OK |
| §7.1 13 因子 | 选股 | BULL 选股累计 -103% | 失效 |

### 5.3 改进建议（按优先级）

#### 优先级 P0: 修复 BULL 状态识别滞后
```yaml
# 缩短迟滞和冷却
min_duration: 3 → 2     # 状态确认从 3 日缩短到 2 日
cooldown_days: 5 → 3     # 冷却期从 5 日缩短到 3 日
crash_5d_drawdown: -0.08 → -0.06  # CRASH 触发更敏感
```
理由：2018-2020 期间状态转换频繁，3 日迟滞错过入场时机。

#### 优先级 P0: 修复移动止盈 vol_adj 公式
当前 `vol_adj = clip(target_vol / max(realized_vol, 0.08), 0.6, 1.5)`
- 在 realized_vol=0.15 时 vol_adj = 1.0（无调整）
- 在 realized_vol=0.30 时 vol_adj = 0.5（缩小 k）
- 这与 spec §6.1 "k = base × vol_adj" 一致，但 k 缩小反而让止盈更近，更易触发
- **建议**: 反转逻辑——低波动时 k 应该缩小（更紧），但 spec 反了

#### 优先级 P1: 加入市场宽度实际计算
spec §6.1 要求 breadth = 沪深 300 成份股中站上 MA60 的比例
当前简化：沪深 300 指数站上 MA60 → breadth=1.0，否则 0.0
**建议**: 加载沪深 300 全部 300 只成分股数据计算真实比例

#### 优先级 P1: 实现 §7.3 IC 加权合成
spec §7.3 明确要求 "IC 加权（基于滚动 120 日 IC），每月更新权重"
当前实现：固定权重 × 状态自适应（spec §7.3 第二段）
**建议**: 计算因子 IC，按月更新权重

#### 优先级 P2: 测试其他区间
- 2015-2017 牛市（测试 BULL 表现）
- 2020-2021 牛市（测试状态切换）
- 2022 年熊市（测试 CRASH 表现）

### 5.4 不建议继续调优的方向

- **继续收紧 BULL 阈值**（V3 教训）
- **大幅减少 BULL 持仓数到 4 以下**（V3 教训）
- **缩短移动止盈 k 到 2.0 以下**（V3 教训）

### 5.5 最终建议

| 方案 | 描述 | 风险 |
|------|------|------|
| **A. 暂停调优，回归策略分析** | 5 次调优已证明 spec 在该区间失效，应回归 spec §6/§7 实施 | 低 |
| **B. 修复 P0 后再调优** | 先修 BULL 状态滞后 + vol_adj 公式，再做下一轮 | 中 |
| **C. 更换测试区间** | 测试 2020-2021 牛市，看策略在趋势市的真实表现 | 中 |

**推荐方案 A + B 组合**：先实施 P0 修复（状态识别 + vol_adj 公式），再决定是否继续调优。

## 7. 结论

### 7.1 调优结论
1. **5 次调优未达成 spec 4/4 目标**
2. **V1 (spec) 是 5 版本中相对最优**
3. **BULL 状态亏损是核心问题，无法通过参数调优解决**
4. **V2 的回撤（-13.76%）满足 ≤15% 目标，是唯一回撤达标的版本**

### 7.2 策略结论
1. **ATOS spec 设计完整**（5 层风控、4 状态、13 因子、自适应）
2. **回测引擎实现到位**（T+1、涨跌停、5 费、滑点、整手）
3. **spec 在 2018-2020.4 中证 1000 表现不佳**（-6% ~ -8% 年化）
4. **BULL 状态识别滞后 + 移动止盈在低波中失效是核心问题**

### 7.3 建议优先级
- P0: 修复 BULL 状态识别滞后（min_duration 3→2, cooldown 5→3）
- P0: 修复移动止盈 vol_adj 公式（反转逻辑）
- P1: 实现真实 market_breadth
- P1: 实现 §7.3 IC 加权合成
- P2: 测试其他区间验证策略

### 7.4 风险提示
- **本测试区间不友好**（2 年熊市 + 1 年震荡）
- **BULL 状态在测试区间被反复误判**
- **建议下一步修复 P0 后再做新一轮调优**
