# 数据目录

> 策略开发与回测使用的本地数据（来源：金玥数据）。两个子目录采用不同的组织方式，按需取用。

---

## 0. 代码生成用途（**重要**）

**本文件是 backtest 代码生成器的核心输入之一**——不是给人翻阅的纯文档，而是给**生成回测代码的 LLM** 读的"数据契约"。

### 0.1 三份输入的关系

生成回测代码时，**必须**把以下 3 份内容**一起**喂给 LLM（缺任一份都会出问题）：

| # | 输入 | 角色 | 提供什么 |
|---|---|---|---|
| 1 | **本文件**（`data/README.md`） | **数据契约** | 字段定义 / 类型 / 读取约定 / 过滤规则 / 字段 → 模块映射（§7） |
| 2 | `subject/CLAUDE.md` | **代码生成 spec** | 要生成哪些文件 / 每个文件的接口契约 / LLM 调用点 / 公共 CLI 参数（§1.4） |
| 3 | `subject/<auto_name>/strategy/<auto_name>_v<N>.md`（运行期） | **业务逻辑** | 入场/出场信号 / 因子 / 止损止盈 / 仓位控制（Part A + Part B） |

**为什么三者缺一不可**：

- **缺本文件** → 生成的 `data_loader.py` 不知道字段名 / 类型 / 读取约定 / 过滤规则；`signal_generator.py` 不知道有哪些因子字段可用
- **缺 `subject/CLAUDE.md`** → 不知道要生成 `signal_generator.py` / `portfolio.py` / `llm_report.py` 等文件；不知道 LLM 提取 + LLM 自评机制；不知道 `--universe` / `--start-date` / `--end-date` 这套 CLI 公共参数
- **缺策略 .md** → 不知道具体信号的入场/出场条件 / 因子权重 / 止损止盈 / 持仓比例——LLM 提取参数时无从下手

### 0.2 数据集（universe）来源 — **不在本文件**

`HS300` / `CSI1000` / `CYB_STAR_50` 这几个股票池 list **不**在本文件，而在 `data/config.py`（项目根）。本文件**只**描述行情数据；股票池 / 时间范围由 `subject/CLAUDE.md §1.4` 的 CLI 公共参数控制。

### 0.3 字段 → 模块映射（详见 §7）

§7 给出了**38 个字段 → 5 个 backtest 模块**的映射表。生成代码时，LLM 应**严格按映射表**决定每个字段在哪个模块里被使用、是否参与过滤、是否参与信号计算。

## 1. 目录结构总览

```
data/
├── data-by-day/            # 横截面：一天一个文件，按年份分目录
│   ├── 2018/
│   ├── 2019/
│   ├── 2020/
│   ├── 2021/
│   ├── 2022/
│   ├── 2023/
│   ├── 2024/
│   ├── 2025/
│   └── 2026/
│       └── YYYY-MM-DD_金玥数据.csv
│
├── data-by-stock/          # 时间序列：一只股票一个文件（扁平）
│   └── XXXXXX_金玥数据.csv
│
└── README.md               # 本文档
```

| 子目录 | 视角 | 路径模式 | 适合的场景 |
|---|---|---|---|
| `data-by-day/` | 横截面：每天全市场 | `data-by-day/{YYYY}/{YYYY-MM-DD}_金玥数据.csv` | 截面因子计算、选股、当日诊断、Phase 2 组合回测 |
| `data-by-stock/` | 时间序列：单只股票全历史 | `data-by-stock/{XXXXXX}_金玥数据.csv` | 单只 K 线回放、长时间窗口指标、跨年回测、Phase 1 信号验证 |

**`data-by-day/` 按年份分目录的原因**：超过 8 年数据若平铺到单层目录,文件数 2200+,在 Windows 文件浏览器 / 部分 IO 库上浏览 / glob 性能都受影响;按年份分目录后单层最多 250 文件,平衡浏览体验与目录深度。

## 2. 共同 Schema（38 列）

两个子目录的字段完全一致，共 38 列。详细字段定义见下表。

### 2.1 标识 / 元信息（4 列）

| 字段 | 类型 | 说明 |
|---|---|---|
| `日期` | string `YYYY-MM-DD` | 交易日 |
| `代码` | string 6 位 | **纯数字，无交易所后缀**（如 `000001`） |
| `名称` | string | 股票名称，**含全角空格**（如 `万  科Ａ`） |
| `所属行业` | string | 行业分类 |

### 2.2 行情（6 列）

`开盘价` / `最高价` / `最低价` / `收盘价`（元）/ `前收盘价`（元）/ `振幅%`

### 2.3 成交（4 列）

`成交量（股）` / `成交额（元）` / `换手率`（%）/ `量比`（倍）

### 2.4 涨跌相关（6 列）

`涨幅%` / `3日涨幅%` / `6日涨幅%` / `10日涨幅%` / `25日涨幅%` / `是否涨停`（`是`/`否`）

### 2.5 股本 / 市值（4 列）

`总股本（股）` / `流通股本（股）` / `总市值（元）` / `流通市值（元）`

### 2.6 估值（3 列）

`滚动市盈率` / `市净率` / `滚动市销率`

### 2.7 均线（7 列）

`5日线` / `10日线` / `20日线` / `30日线` / `60日线` / `120日线` / `250日线`（元，简单移动平均）

### 2.8 状态（4 列）

`是否ST`（`是`/`否`）/ `是否融资融券`（`是`/`否`/空）/ `上市时间`（`YYYY-MM-DD`）/ `退市时间`（未退市为 **`-`**，单字符减号）

## 3. `data-by-day/`（横截面）

- **路径**：`data/data-by-day/{YYYY}/`（**按年份分目录**,**不**是扁平)
- **文件命名**：`YYYY-MM-DD_金玥数据.csv`
- **覆盖年份**：2018 ~ 2026（9 个年份目录,2018-01-02 ~ 2026-05-14,**约 8.4 年**)
- **每年文件数**：~243（240 ~ 244,因春节/节假日略有差异)
- **总文件数**：~2267
- **单文件大小**：~1.5 ~ 1.6 MB
- **每文件记录数**：约 4000-5000 行（覆盖全 A 当日所有股票）

**目录示例**：

```
data-by-day/
├── 2018/   ← 2018-01-02_金玥数据.csv ... 2018-12-28_金玥数据.csv  (243 文件)
├── 2019/   ← 2019-01-02_金玥数据.csv ... 2019-12-31_金玥数据.csv  (244 文件)
├── 2020/   ← ...  (243 文件)
├── 2021/   ← ...  (243 文件)
├── 2022/   ← ...  (242 文件)
├── 2023/   ← ...  (242 文件)
├── 2024/   ← ...  (242 文件)
├── 2025/   ← ...  (243 文件)
└── 2026/   ← 2026-01-05_金玥数据.csv ... 2026-05-14_金玥数据.csv  (81 文件,年内未结束)
```

**读取路径构造**:

```python
from pathlib import Path
year = "2026"
date = "2026-05-14"
csv_path = Path(f"data/data-by-day/{year}/{date}_金玥数据.csv")
# → data/data-by-day/2026/2026-05-14_金玥数据.csv
```

**注意**:
- **可独立覆盖 8+ 年牛熊周期**——`data-by-day/` 自 2018-01 起,完整覆盖 2018 熊末 / 2019-2020 牛 + 疫情 / 2021 牛尾 / 2022 熊 / 2023-2024 震荡 / 2025 反弹 / 2026-Q1 续涨等多个完整市场状态;Phase 2 评估（`../CLAUDE.md §5.3` "5 年以上样本 + 完整牛熊周期"）可直接用 `data-by-day/` 主测,不再依赖 `data-by-stock/` 兜底
- 适合做"按日截面选股"型策略、Phase 2 组合回测、因子横截面打分
- 9 个年份目录互不交叉,glob 时**必须**指定 `data-by-day/{year}/*` 或 `data-by-day/*/*`（**不**能只 glob 一层 `data-by-day/*`,否则只匹配到 9 个年份子目录,匹配不到 CSV）

## 4. `data-by-stock/`（时间序列）

- **路径**：`data/data-by-stock/`
- **命名格式**：`XXXXXX_金玥数据.csv`（6 位纯数字代码）
- **文件数**：5841
- **覆盖日期**：每只股票从其上市日 ~ 2026-05-14（**最长约 26 年**，如 000001 平安银行自 2000-01-04 起）
- **单文件大小**：~1.6 MB（活跃股）/ 几十 KB（冷门股，如 000003 仅 ~99 KB）
- **每文件记录数**：每只股票的全部历史日线

**注意**：
- **包含北交所股票**（代码以 `92` / `83` 开头，如 `920982` / `830xxx`）——CLAUDE.md §1 已将北交所排除在策略范围外，读数据时需按代码前缀过滤
- **早期均线字段为空**：上市未满 250 日的股票，前 250 日的 `120日线` / `250日线` 等字段为 `NaN`/空（数据源在均线算不出来时留空）
- **早期涨幅字段为空**：上市未满 N 日的 `3日涨幅%` / `6日涨幅%` 等滚动涨幅字段为空
- **退市股**仍保留在目录中（`退市时间` 有值）——按 CLAUDE.md §4 硬约束"新股 / 退市 / 一字板默认跳过"
- 适合做"长周期回测 / 牛熊穿越验证 / 单股深度分析"

---

## 5. 读取时通用处理

两个子目录读取时都需要做的事：

1. **`代码` 补后缀**：6 位数字 → `xxx.SZ` / `xxx.SH`（按交易所映射：`60xxxx` / `68xxxx` → SH；`00xxxx` / `30xxxx` / `20xxxx` → SZ；`92xxxx` / `83xxxx` → BJ）
2. **`名称` 去全角空格**：`万  科Ａ` → `万科A`（`strip()` 或正则 `\s+` → `""`）
3. **`退市时间` 空值**：`-` 视为 `NaT`/空，不当作有效日期
4. **`日期` 转 Timestamp**：`YYYY-MM-DD` 字符串 → `pd.Timestamp`
5. **`是否ST` / `是否涨停` / `是否融资融券`** 转 `bool`：`是` → `True`，`否`/空 → `False`

---

## 6. 数据完整性 / 整体注意事项

- **"穿越牛熊"主测用 `data-by-day/`**（**2018 起的 8+ 年**）+ **`data-by-stock/` 兜底**（最长 26 年）:CLAUDE.md §5.3 要求的 5 年样本外 + 完整牛熊周期,两者都能直接兑现;`data-by-day/` 适合 Phase 2 组合回测,`data-by-stock/` 适合 Phase 1 单股信号验证
- **`data-by-day/` 按年份分目录**：glob 时**必须**两层 `*/*` 模式（`data-by-day/*/*_金玥数据.csv`）;Python 读取时按 `data-by-day/{year}/{date}_金玥数据.csv` 路径构造
- **跨目录代码对齐**：两边的代码都是 6 位纯数字格式,可以直接用代码字段对齐;统一补后缀后再做 join
- **停牌 / 复牌日的数据形态**:未抽样确认,按 CLAUDE.md §4 硬约束约定,停牌日不成交——读数据时注意"成交量=0、收盘价=前收盘价"的停牌特征
- **文件大小一致性**:可作为数据缺失的快速检测指标(同目录内文件大小异常 = 当天/那只股票数据可能不全)

### 7.2 读取约定（`data_loader.py` 必读）

`data_loader.py` 在读取 CSV 后**必须**做的 5 项处理（漏做任一项都会导致回测失真）：

| # | 处理 | 代码片段（伪代码） | 漏做的后果 |
|---|---|---|---|
| 1 | `代码` 补后缀 | `code_to_suffix(code) → "xxx.SZ"` / `"xxx.SH"`（按 §5.1 映射） | 对接交易所 / Tushare / AkShare 时 join 失败 |
| 2 | `名称` 去全角空格 | `name = re.sub(r"\s+", "", str(name))` | 显示 / 日志不一致 |
| 3 | `退市时间` `-` → `NaT` | `退市时间 = pd.to_datetime(退市时间, errors="coerce")`（`-` 自动转 NaT） | 退市过滤条件恒为 True/False |
| 4 | `日期` → `pd.Timestamp` | `日期 = pd.to_datetime(日期)` | 时间索引失效 / 范围过滤报错 |
| 5 | `是否ST` / `是否涨停` / `是否融资融券` → `bool` | `flag.map({"是": True, "否": False, "": False})` | bool 判断条件恒为 True/False |
